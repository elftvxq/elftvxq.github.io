🌐 Internet & Networking Terminology 網路名詞小筆記

Posted on 2025-05-11 In networking Views: Word count in article: 2.2k Reading time ≈ 4 mins.

🔁 TCP（Transmission Control Protocol，傳輸控制協定）

TCP 是一種可靠的資料傳輸協定，它的主要任務是確保資料能正確、完整地送到對方手中。

✋ 三次握手（Three-way Handshake）

TCP 傳輸會在連線前經過「三次握手」來建立穩定的連線。
。這三步驟像是在雙方說：「我準備好了嗎？你也好了嗎？好，那我們開始吧。」

過程簡化如下：

Client：嗨，我要傳資料囉！（SYN）

Server：我準備好了，你準備好了嗎？（SYN-ACK）

Client：我真的準備好了！（ACK）

三次握手完成後，雙方才會開始傳送資料。

✅ 適用場景：
瀏覽網頁

檔案下載

資料庫連線

只要是 需要資料完整、不容出錯的場合，都會用 TCP。

三次握手完成後，資料才會開始傳輸。

✅ 適用場景：

  • 瀏覽網頁
  • 檔案下載
  • 資料庫連線

🔍 進階補充（TCP）

  • 屬於 OSI 模型第 4 層：傳輸層
  • 為 面向連線（connection-oriented）協定
  • 提供封包編號、回應確認、重傳與流量控制
  • 包含擁塞控制、流量控制、檢查碼驗證等機制
  • 常見欄位：Sequence Number、ACK Number、SYN、ACK、FIN

📡 UDP（User Datagram Protocol，用戶資料報協定）

UDP 是一種「輕量、快速、但不保證完整性」的協定，就像把明信片丟進郵筒，不確認是否有收到。

✅ 適用場景：

  • 🎥 看直播
  • 🎮 玩線上遊戲
  • 📡 VoIP 語音通訊

🔍 進階補充（UDP）

  • 同樣是 OSI 第 4 層傳輸層協定
  • 無連線（connectionless），不握手
  • 不保證順序、不重傳、不確認
  • 適合即時性高的應用

哪些場景為什麼用 UDP 而不是 TCP？

因為有些情境中，速度比精確更重要，例如：

看直播（掉一兩個 frame 沒關係）
玩線上遊戲（比起補齊資料，更在乎即時回應）

📣 ICMP（Internet Control Message Protocol）

ICMP 用於傳送網路錯誤訊息與診斷，例如 ping、traceroute 就是透過 ICMP 運作。
透過 terminal 輸入

其實背後就是透過 ICMP 傳送「我可以連到你嗎？」的訊息，然後對方回應。

🔍 進階補充（ICMP）

  • 屬於 OSI 第 3 層（網路層）
  • 常見用途：
    • ping: 發送 Echo Request → 回 Echo Reply
    • traceroute: TTL 遞減觸發 Time Exceeded
  • 用於傳送網路控制訊息，而非應用資料

🌍 DNS（Domain Name System）

DNS 是網際網路的電話簿，負責將網域名稱（如 google.com）轉換成 IP 位址，但機器只認得 IP 位址（如 142.250.72.14）。
當你輸入網址時，DNS 就會幫你查出對應的 IP，像是在說：「喔，google.com 對應的地址是這個。」

🔍 進階補充（DNS）

  • OSI 應用層協定，使用 Port 53
  • 查詢類型：
    • 遞迴查詢（由 Resolver 幫忙找到最終結果）
    • 迭代查詢（DNS Server 分層協助）
  • 常見記錄類型：
    • A, AAAA, CNAME, MX, NS…
  • 有 TTL 快取機制

🗂️ Nameserver（名稱伺服器）

Nameserver 是負責 回應 DNS 查詢請求 的伺服器。每個網域註冊後都需要指定 Nameserver，告訴全世界：

「如果有人想找我的 IP，請問這台伺服器！」

例如：

  • google.com 的 nameserver 負責回答「google.com 是哪個 IP？」

  • 有時候改變網站的主機時，也要修改對應的 nameserver 設定讓用戶知道去哪裡解析該網域。

🔍 進階補充（Nameserver）

  • 屬於 DNS 架構中的「授權名稱伺服器」
  • 管理特定網域的解析記錄
  • 網域設定時需指定其 Nameserver（可指向 Cloudflare 等）

🔗 URL（Uniform Resource Locator，統一資源定位符）

URL 是網路世界的地址，告訴瀏覽器該去哪裡、取什麼資源、怎麼存取。

📌 結構範例

https://www.example.com:443/path/to/page?query=value#section1

🔍 進階補充（URL 結構）

區塊 名稱 說明
https 協定 http / https / ftp…
www.example.com 主機名稱 DNS 會解析成 IP 位址
:443 埠號 預設 http:80 / https:443
/path/... 路徑 網頁或資源位置
?query=value 查詢參數 用來傳資料（GET 請求）
#section1 錨點 網頁內部跳轉

🎯 範例解析

https://www.youtube.com/watch?v=dQw4w9WgXcQ

  • 協定：HTTPS
  • 主機：YouTube
  • 路徑：/watch
  • 查詢參數：v = dQw4w9WgXcQ（影片 ID）

🧠 總結表格

名詞 作用 特性
TCP 傳送穩定資料 有連線、可靠、三次握手、重傳、擁塞控制
UDP 快速傳輸資料 無連線、不保證資料到達、適合即時應用
ICMP 診斷與錯誤通訊 用於 ping、錯誤回報、屬於網路層
DNS 將網址轉成 IP 網路的電話簿、支援快取、多種記錄類型
Nameserver 回答 DNS 查詢 管理授權網域、與 DNS 配合
URL 指定網路資源位置 協定、主機、路徑、參數、錨點等結構資訊
